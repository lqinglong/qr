QR Code Scanner Privacy Policy

Last Updated: May 22, 2025

1. Scope of Application

This policy applies to all functions and services of the "QR Code Scanner" app (hereinafter referred to as "this app"). This app is a localized tool that does not connect to the internet, does not collect, does not store, and does not share any user information.

2. Permissions

This app requires the following permissions to function:

  • Camera Permission: Used to scan QR code content. All image processing is completed locally on the device, and no image data is stored or uploaded.
  • Internet Permission: Only used to load the online version of this privacy policy when the user actively clicks "View Privacy Policy" (if applicable).

3. Data Collection and Use

QR Code History Explanation:
This app provides a scan history feature. The content of the most recently scanned QR codes is stored only in the device's cache files to facilitate users' viewing of their history. These data:
  • Are stored only on your local device
  • Are not encrypted (since they do not contain sensitive information)
  • Are not uploaded to any server
  • Are automatically cleared when the app is uninstalled
  • Can be manually deleted via the "Clear History" function within the app

Other Data:

  • This app does not collect personal information (such as name, device ID, location, etc.)
  • The content of scanned QR codes is only temporarily processed in the device's memory and is not saved persistently or transmitted to a server
  • This app does not integrate any third-party SDKs (such as advertising or analytics tools) and does not need to share data

4. Data Security

  • All permissions are granted by the user, and you can disable them at any time through your device settings (disabling the camera permission will cause the scanning function to fail)
  • History data is stored only within the app's sandbox, and other apps cannot access it
  • This app complies with Google Play's target API level requirements (Android 14 or higher), ensuring system security

5. Policy Updates

If there are any changes to the policy, we will notify you through in-app announcements or update logs. Continued use is considered acceptance of the revised policy.
